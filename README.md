# CircleDrawer

세 점을 지나는 외접원을 실시간으로 그리는 MFC Dialog 기반 애플리케이션입니다.

---

## 기능

| 기능 | 설명 |
|------|------|
| 점 클릭 | 좌클릭 3번으로 점 배치, 3번째 클릭 시 외접원 자동 표시 |
| 4번째 클릭 | 점 추가 없음 (무시) |
| 드래그 | 점 원을 클릭·드래그하면 외접원 실시간 재계산 |
| 점 원 반지름 | 오른쪽 입력란에서 조절 (1~100px) |
| 정원 선 두께 | 오른쪽 입력란에서 조절 (1~20px) |
| 좌표 표시 | P1 / P2 / P3 좌표를 오른쪽 패널에 실시간 표시 |
| 초기화 | 모든 점과 원 삭제, 초기 상태로 복귀 |
| 랜덤 이동 | 3개 점을 0.5초 간격으로 10회 랜덤 이동 (백그라운드 스레드, UI 멈춤 없음) |

---

## 실행 예시

**1. 점 3개 클릭 → 외접원 표시**
```
① 빈 영역 아무 곳이나 클릭  →  P1 배치
② 다른 곳 클릭              →  P2 배치
③ 또 다른 곳 클릭           →  P3 배치 + 외접원 자동 표시
```

**2. 점 드래그**
```
① 점 원 위에 마우스 올리기
② 클릭한 채로 드래그
   → 마우스를 움직이는 동안 외접원이 실시간으로 따라 움직임
```

**3. 랜덤 이동**
```
① 세 점이 배치된 상태에서 [랜덤 이동] 클릭
② 0.5초마다 세 점이 랜덤 위치로 이동 (총 10회)
③ 이동 중에도 UI는 반응 유지 (스레드 분리)
```

**4. 초기화**
```
[초기화] 클릭 → 모든 점·원 삭제, 처음부터 다시 입력 가능
```

## 구현 핵심

### 원 그리기 (Ellipse 금지)

- **채워진 점 원** — 수평 스캔라인: y 한 줄씩 `MoveTo → LineTo`로 채움
- **빈 외접원** — 파라메트릭 근사: 360°를 720등분한 꼭짓점을 `LineTo`로 연결

### 외접원 수학

세 점의 수직이등분선 교점으로 중심을 계산합니다.

```
D = 2 * (ax*by - ay*bx)
중심 오프셋 ux, uy = 벡터 내적 / D
반지름 r = √(ux² + uy²)
```

세 점이 일직선이면 원을 그리지 않습니다.

### 깜빡임 방지 (더블 버퍼링)

오프스크린 비트맵(`CDC memDC`)에 먼저 전부 그린 뒤 `BitBlt`로 한 번에 복사합니다.  
`InvalidateRect(&m_drawRect, FALSE)` — `FALSE`로 배경 지우기를 생략합니다.

### 랜덤 이동 스레드

```
[랜덤 이동 버튼]
      │
      ▼
CreateThread → DoRandomMove (워커 스레드)
                  │  Sleep(500ms) × 10회
                  │  EnterCriticalSection
                  │  m_pts 랜덤 변경
                  │  LeaveCriticalSection
                  └─ PostMessage(WM_RANDOM_UPDATE)
                              │
                              ▼
                    OnRandomUpdate (UI 스레드)
                    ComputeCircumscribed + Redraw
```

UI 스레드와 워커 스레드가 `m_pts`를 공유하므로 `CRITICAL_SECTION`으로 보호합니다.  
UI 업데이트는 반드시 UI 스레드에서만 수행해야 하므로 `PostMessage`로 신호만 전달합니다.

## 빌드 환경

|------|------|
| IDE | Visual Studio 2022 |
| 프레임워크 | MFC (Dynamic) |
